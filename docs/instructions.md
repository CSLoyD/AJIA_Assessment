# Project Instructions

Stakeholder instructions given at project kickoff, recorded verbatim for reference during
implementation and review.

1. No emoji anywhere (UI, code, commit messages, docs).
2. All instructions and specs live under `docs/`.
3. `AGENTS.md` and other AI-assistant instruction files must not be committed; they are
   gitignored.
4. Always ask clarifying questions before starting implementation.
5. Give a clear recommendation on what stack/tools to use, do not leave it open-ended.
6. Include automated testing (unit tests or equivalent).
7. Containerization is required: a Docker Compose setup must be provided.
8. Minimize code comments. Favor readable, self-explanatory code and structure. Add a
   comment only when it is genuinely needed, and keep it short.

See [specs/2026-07-23-collaborative-doc-editor-design.md](specs/2026-07-23-collaborative-doc-editor-design.md)
for the product/technical design derived from these instructions plus the task brief.
